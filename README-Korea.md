# RFrontEnd

기존 구현의 코드를 전혀 재사용하지 않고 HTML5/CSS3/TypeScript/React에 상응하는
기능을 처음부터 개발하는 여러 프로젝트를 묶는 상위(부모) 리포지토리입니다.

📖 다른 언어: [日本語](README-Japan.md) / [English](README-English.md) /
[中文](README-Chinese.md) / [한국어](README-Korea.md) / [Español](README-Spain.md) /
[Français](README-France.md) / [Deutsch](README-Germany.md) / [Italiano](README-Italy.md) /
[Русский](README-Russia.md) / [العربية](README-Arabic.md) —
다른 프로젝트로의 도입은 **[PORTING.md](PORTING.md)** 한 장으로 끝납니다.

## 산하 프로젝트

| 프로젝트 | 역할 | 리포지토리 |
|---|---|---|
| RHTML | HTML5에 상응하는 파서/DOM 구현 | [RTHML](https://github.com/aon-co-jp/RTHML) |
| RCSS | CSS3에 상응하는 파서/캐스케이드 계산 | [RCSS](https://github.com/aon-co-jp/RCSS) |
| RTypeScript | TypeScript에 상응하는 구현 | [RTypeScript](https://github.com/aon-co-jp/RTypeScript) |
| RJSON | JSON 처리 | [RJSON](https://github.com/aon-co-jp/RJSON) |
| RReact | React에 상응하는 컴포넌트 모델 | [RReact](https://github.com/aon-co-jp/RReact) |

서버 측 실행 기반은 [RPoem](https://github.com/aon-co-jp/RPoem)(Poem을
처음부터 재개발)과 [RCosmo](https://github.com/aon-co-jp/RCosmo)(구
open-runo)가 담당합니다.

자세한 개발 방침, 개발 순서, 아키텍처는 `CLAUDE.md`를 참조하세요.
